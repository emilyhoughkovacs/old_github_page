Hi there, I'm Emily.

My Background.

Originally a mathematics major at Barnard College, I remember the exact moment when I realized intuitively how useful computing could be. Sitting in my dorm lounge, endlessly plugging in data points one by one to my TI-89 calculator to practice matrix multiplication, I realized I could use my time a lot better if I knew how to code something to do the same operations for me. This was the first in a series of 'aha' moments that led me down a path exploring computer science, data analytics, data visualization, and data science. Outside of my professional life, I am a dog-lover, ramen-explorer, Nicki Minaj fan and wilderness-canoe-camper.

My Next Adventure.

Right now I'm actively seeking new opportunities! My ideal next position would allow me to learn more about machine learning models, algorithms, and data science best practices, while allowing me to flex my killer analytical, communication, and reporting skills. I'd also like a chance to work on beautiful, informative data visualizations. What's most important to me, however, is company culture. I've been lucky to work at places where my colleagues are excited to come to work each day because they love their product and are proud of their company's mission and values.

Tools I Love.

I've worked primarily in Python. For data analysis I use pandas, but in the past I've used R. I've toyed around with Flask, Redis and Celery when I built a web app that called a Python subprocess. I am comfortable with MySQL, PostgreSQL, and Hive for Hadoop. I'm familiar with the gspread API for Python. For smaller analyses, I can whip out an Excel or Google Spreadsheets pivot table. To quickly summarize data I prefer Google Charts, but matplotlib (and mpld3, a package that converts matplotlib graphs to d3) is more dynamic. I'm also an experienced Tableau administrator. In school, I was classically trained in Java and C.
